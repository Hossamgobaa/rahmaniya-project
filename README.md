# دليل الرحمانية — التوثيق الكامل

## 📁 هيكلة المشروع
```
/index.html              صفحة الموقع العامة (للزوار)
/manifest.json           إعدادات تثبيت التطبيق (PWA)
/sw.js                   Service Worker (العمل بدون إنترنت)
/admin/index.html        لوحة الإدارة (محمية بتسجيل دخول)
/admin/admin.js          منطق لوحة الإدارة
/assets/css/style.css    كل التنسيقات (مشتركة)
/assets/js/common.js     البيانات والدوال المشتركة بين الموقع واللوحة
/assets/js/app.js        منطق الموقع العام
/assets/icons/           أيقونات التطبيق (192px, 512px)
/firebase/firebase-config.js   بيانات الاتصال بقاعدة البيانات
/firebase/firestore.rules      قواعد الأمان الموصى بها
```

---

## 1) ربط Firebase (الخطوة الأهم)

### أ) إنشاء المشروع
1. روح على **console.firebase.google.com** وسجّل دخول بحساب جوجل
2. **Add project** → اكتب اسم (مثلًا "دليل-الرحمانية") → كمّل الخطوات

### ب) تفعيل قاعدة البيانات
1. من القائمة الجانبية: **Build → Firestore Database → Create database**
2. اختار **Start in production mode** (مش test mode هذه المرة، لأننا هنحط قواعد أمان حقيقية في الخطوة 3)
3. اختار أقرب موقع سيرفر، ودوس **Enable**

### ج) جلب بيانات الاتصال
1. أيقونة الترس ⚙️ → **Project settings**
2. نزّل لـ **Your apps** → دوس أيقونة **`</>`** (Web) → سجّل اسم التطبيق
3. هيظهر لك كود فيه `firebaseConfig` — افتح ملف **`firebase/firebase-config.js`** وحط كل قيمة في مكانها بدل `"ضع_..._هنا"`

### د) تفعيل تسجيل دخول الأدمن (مهم جدًا)
1. من القائمة الجانبية: **Build → Authentication → Get started**
2. تبويب **Sign-in method** → فعّل **Email/Password**
3. تبويب **Users** → **Add user** → حط إيميلك وكلمة سر قوية (دي بيانات دخولك للوحة الإدارة)

### هـ) رفع قواعد الأمان
1. **Firestore Database → تبويب Rules**
2. امسح المحتوى الموجود، وافتح ملف **`firebase/firestore.rules`** عندك، انسخ المحتوى بالكامل، الصقه، ودوس **Publish**

---

## 2) تشغيل المشروع محليًا
المشروع مكوّن من ملفات HTML/JS عادية، ميحتاجش تثبيت أي حاجة. بس Firebase وService Worker مش بيشتغلوا صح من `file://` مباشرة، فلازم تشغّله من خلال سيرفر محلي بسيط:

- لو عندك Python: افتح Terminal في مجلد المشروع واكتب: `python3 -m http.server 8000` وبعدين افتح `http://localhost:8000`
- أو استخدم إضافة "Live Server" في VS Code

---

## 3) النشر (Hosting) — مجاني بالكامل
**Firebase Hosting** هو الأنسب لأنه نفس مشروعك:
1. ثبّت أداة Firebase (محتاج Node.js مثبت على جهازك): `npm install -g firebase-tools`
2. ملف `firebase.json` جاهز بالفعل في المشروع (معمول عليه ضبط مسبق)، فقط من مجلد المشروع: `firebase login` ثم `firebase use --add` (اختار مشروعك)
3. `firebase deploy` — هذا الأمر هيرفع الموقع **وقواعد الأمان معًا** في خطوة واحدة (بفضل `firebase.json`)
4. هتاخد رابط زي: `https://your-project.web.app` — ده موقعك الحقيقي على الإنترنت

---

## 4) تحويل الموقع لتطبيق APK
بعد ما يكون الموقع منشور على رابط حقيقي (من خطوة 3):
1. روح على **pwabuilder.com**
2. حط رابط موقعك واضغط Enter
3. هيفحص الموقع (هيلاقي manifest.json و sw.js جاهزين بالفعل)
4. دوس **Package for stores → Android** وحمّل ملف APK جاهز للتثبيت أو النشر على Google Play

ملاحظة: على iPhone، المستخدم بيثبّت الموقع كتطبيق مباشرة من Safari (مشاركة → "إضافة إلى الشاشة الرئيسية") بدون احتياج APK، وده مدعوم تلقائيًا بفضل manifest.json.

---

## 5) دومين مخصص (اختياري)
لو شريت دومين (مثل rahmaniah-guide.com) من أي شركة (Namecheap, GoDaddy...):
1. Firebase Hosting → **Add custom domain**
2. اتبع التعليمات (هتحتاج تضيف سجلات DNS عند شركة الدومين)

---

## 6) الأمان — أقصى مستوى ممكن 🔒

### أ) تسجيل الدخول (Authentication)
- مفعّل بطريقة **Email/Password فقط** — مفيش أي طريقة تسجيل مفتوحة (Google/فيسبوك/إلخ) ومفيش أي فورم "تسجيل حساب جديد" في الكود كله، فمستحيل لحد يعمل حساب لنفسه.
- الحساب الوحيد هو اللي تنشئه إنت بنفسك من **Authentication → Users → Add user**.

### ب) قواعد قاعدة البيانات (Firestore Rules) — Read Only للزوار
ملف `firebase/firestore.rules` دلوقتي بيطبّق:
- **القراءة (Read)**: مسموحة للجميع (عشان الزوار يقدروا يتصفحوا الموقع)
- **أي كتابة (إضافة/تعديل/حذف)**: ممنوعة تمامًا إلا لحساب الأدمن المسجّل دخول — **بدون أي استثناء واحد**.

⚠️ **خطوة مهمة لازم تعملها**: افتح `firebase/firestore.rules` وغيّر:
```
request.auth.token.email == "ضع_إيميل_الأدمن_هنا@example.com"
```
لإيميلك الحقيقي (نفس إيميل حساب الأدمن اللي أنشأته). من غير الخطوة دي، حتى حسابك الأدمن مش هيقدر يعدّل أي بيانات.

### ج) صفحة الإدارة منفصلة ومحمية بالكامل
- `admin/index.html` صفحة مستقلة 100% عن الموقع العام.
- البيانات مش بتتحمّل أو تتجاب من القاعدة **إلا بعد تأكيد Firebase إن فيه تسجيل دخول حقيقي** (مش بس إخفاء بالـ CSS) — يعني حتى لو حد فتح أدوات المطور، مفيش بيانات إدارية توصله أساسًا.
- غير المسجّل دخول يشوف فورم الدخول فقط، ولا أي محتوى آخر.
- مُستبعدة من نتائج البحث (`robots.txt` + `noindex`).

### د) الموقع العام Read-Only بالكامل
- ملف `assets/js/app.js` (منطق الموقع العام) **لا يحتوي على أي دالة كتابة لقاعدة البيانات من الأساس** — كل دوال الإضافة/التعديل/الحذف موجودة فقط في `admin/admin.js`. حتى لو حصل تلاعب في كود المتصفح، مفيش طريقة فعلية يضيف بها حد بيانات.

### هـ) تقييد مفتاح API على دومينك فقط
1. روح على **console.cloud.google.com** (نفس مشروع Firebase) → **APIs & Services → Credentials**
2. دوس على مفتاح API بتاعك (هيكون له نفس اسم مشروعك)
3. تحت **Application restrictions** اختار **HTTP referrers** وحط دومينك (مثل `https://rahmaniya-guide.web.app/*` و دومينك المخصص لو عندك)
4. حفظ — دلوقتي المفتاح ميشتغلش من أي موقع تاني غير بتاعك حتى لو حد نسخ الكود

### و) HTTPS تلقائي
**Firebase Hosting بيشغّل HTTPS تلقائيًا لكل المواقع بدون أي إعداد إضافي** — رابطك (`https://your-project.web.app`) هيكون مشفّر دايمًا من أول يوم.

### ز) التجهيز لتطبيق Android (WebView)
- مفيش أي كتابة في Firestore بدون تسجيل دخول (مضمون من قواعد الأمان، مش بس من كود الواجهة)
- **مهم تفهمها**: قيم `firebase/firebase-config.js` (مثل apiKey) **ليست كلمة سر** — هي معرّفات عامة مصمّمة من جوجل للتضمين في كود العميل (نفس مبدأ أي تطبيق موبايل حقيقي). الحماية الفعلية مش من "إخفاء" هذه القيم، لكن من نقطتين (ب) و(هـ) فوق. وبالتالي تطبيق الـ WebView هيكون آمن بنفس درجة الموقع تمامًا.

### ح) نسخ احتياطي لقاعدة البيانات
- لو مشروعك على خطة **Blaze** (الدفع حسب الاستخدام، فيها مستوى مجاني سخي برضو): فعّل **Firestore → Backups → Schedule a backup** من Firebase Console مباشرة (نسخ احتياطي تلقائي يومي/أسبوعي مع إمكانية استرجاع بضغطة).
- لو على الخطة المجانية (Spark): تقدر تعمل نسخة احتياطية يدوية بأمر واحد عبر `gcloud` (يحتاج تثبيت Google Cloud CLI):
  ```
  gcloud firestore export gs://اسم-باكت-التخزين-بتاعك
  ```
  وللاسترجاع: `gcloud firestore import gs://اسم-الباكت/اسم-النسخة`

## 7) إضافة أقسام/بيانات جديدة
كل حاجة من لوحة الإدارة (`admin/index.html`) — مفيش حاجة تحتاج تعديل كود:
- تبويب **"إدارة الأطباء"**: إضافة/تعديل/حذف الدكاترة
- تبويب **"كل الأقسام"**: الصيدليات، المستشفيات، المطاعم، السوبر ماركت، المحلات، التوصيل، الطوارئ، **الإعلانات**
- تبويب **"الأقسام المخصصة"**: لإنشاء أقسام جديدة بالكامل (مثل "إعلانات الوفيات") بأيقونة ولون وصورة غلاف خاصة بيها

---

## 8) الربح من الإعلانات
- **إعلانات محلية (شغالة فورًا)**: من تبويب "كل الأقسام" → "الإعلانات" → أضف صورة + اسم المعلن + رقم/رابط، وتظهر فورًا في الصفحة الرئيسية لكل الزوار
- **Google AdSense (ربح تلقائي حقيقي)**: محتاج موقعك يكون منشور فعليًا (خطوة 3) أولًا، وبعد التقديم والموافقة من جوجل على adsense.google.com، حط `publisherId` و `adSlotId` في ملف `firebase/firebase-config.js`

---

## 9) قائمة ما تم بناؤه ✅ وما هو مُجهّز للمستقبل 🔜
✅ قاعدة بيانات Firestore لكل الأقسام + رفع صور
✅ لوحة إدارة كاملة (إضافة/تعديل/حذف + إنشاء أقسام جديدة بدون كود)
✅ بحث فوري بالاسم/التخصص
✅ خريطة فعلية + تحديد موقع بالضغط + فتح الاتجاهات + اتصال مباشر
✅ **ترتيب حسب الأقرب فعليًا** (بالمسافة الحقيقية GPS) — زر "الأقرب لي" في كل صفحات القوائم
✅ المفضلة + واتساب + مشاركة
✅ **سلايدر إعلانات تلقائي** (يتغيّر كل 5 ثواني) يدعم الصور والفيديو (رابط مباشر أو يوتيوب)
✅ **تثبيت إعلان في الأعلى** + **جدولة بتاريخ بداية/نهاية** + **عداد مشاهدات وضغطات حقيقي**
✅ مكان AdSense جاهز
✅ تسجيل دخول حقيقي بدون كلمة سر في الكود (Firebase Authentication)
✅ PWA (Manifest + Service Worker + يعمل بدون إنترنت)
✅ صفحات قانونية (عدّل نصوصها من `assets/js/app.js`)

🔜 (المرحلة الجاية المقترحة): تقييمات نجوم بمنع التكرار وتعليقات المستخدمين، قسم وظائف/عقارات/أخبار محلية كأقسام مخصصة جاهزة، فيديو عمودي بنمط تيك توك للإعلانات
